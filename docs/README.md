# PaperDB Docs Index

This folder is the written source of truth for the PaperDB monorepo.

Use it in this order:

1. [01_SYSTEM_DOCUMENTATION.md](01_SYSTEM_DOCUMENTATION.md) for the current architecture and service boundaries.
2. [02_GAPS_AND_IMPROVEMENTS.md](02_GAPS_AND_IMPROVEMENTS.md) for the remaining gaps and the next practical fixes.
3. [implementation_plan.md](implementation_plan.md) for the executed milestone plan.
4. [PRODUCTION_READINESS.md](../PRODUCTION_READINESS.md) for deployment and ops confidence.
5. [progress.md](progress.md) for the live, line-by-line status tracker.

Supporting strategy docs:

- [03_SUPABASE_NEON_REPLICA_PLAN.md](03_SUPABASE_NEON_REPLICA_PLAN.md) — long-term product direction.
- [04_SERVER_DEPLOYMENT_AND_ADOPTION.md](04_SERVER_DEPLOYMENT_AND_ADOPTION.md) — deployment models and user value.
- [05_PRD_V1_UNIVERSAL_PAPERDB.md](05_PRD_V1_UNIVERSAL_PAPERDB.md) — V1 requirements and release targets.

The docs deliberately separate three layers:

- what the code does now,
- what still needs work,
- what the product should become next.
